# Yanova Solutions — Website

Corporate website for **Yanova Solutions**, an Algerian cybersecurity & networking startup (est. 2025).
Built as a single deployable Node app: a **React (Vite)** frontend served by an **Express** backend, with a
bilingual (🇫🇷 FR / 🇬🇧 EN) public site, a contact form, a blog, and a private admin panel.

## Features

- **Bilingual public site** (French / English) with a language switcher.
- **Product showcase**: Cerberus Pi (flagship IDS/IPS), Yanova OS, ReconOne, and the Yanova NAS Server.
- **Services**, **About / team**, and an animated cybersecurity-themed hero.
- **Contact form** — submissions are stored in the database.
- **Blog** — public list + article pages, Markdown content, bilingual.
- **Admin panel** at `/admin/login`:
  - View, mark read/unread, and delete contact messages.
  - Create, edit, publish/unpublish, and delete blog posts (bilingual).
- **SQLite** storage (single file, zero configuration).
- JWT-based admin authentication with bcrypt-hashed password and rate limiting.

## Tech stack

| Layer     | Tech                                              |
|-----------|---------------------------------------------------|
| Frontend  | React 18, React Router, Vite, Tailwind CSS, lucide-react |

## Getting started (local)

```bash
# 1. Install dependencies
npm install

# 2. (optional) add a couple of starter blog posts
npm run seed

# 3a. Development — Vite (5173) + API (3000) with hot reload
npm run dev
#    → open http://localhost:5173

# 3b. Production-style — build then serve everything from Express
npm run build
npm start
#    → open http://localhost:3000
```

## Admin access

- URL: **`/admin/login`**
- Credentials come from environment variables (`ADMIN_USERNAME`, `ADMIN_PASSWORD`).
- Configured credentials:
  - Username: `admin@yanovasolutions.tech`
  - Password: `adminYanova!2026`
- **Change `JWT_SECRET`** (and rotate the password) before/after going live. On Hostinger,
  set these as environment variables in the Node.js app panel — see [DEPLOYMENT.md](./DEPLOYMENT.md).

Generate a fresh JWT secret:

```bash
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```

## Configuration (`.env`)

| Variable         | Description                                   |
|------------------|-----------------------------------------------|
| `PORT`           | Port the server listens on (Hostinger sets this automatically). |
| `NODE_ENV`       | `production` in production.                    |
| `ADMIN_USERNAME` | Admin login username.                         |
| `ADMIN_PASSWORD` | Admin login password (hashed at startup).     |
| `JWT_SECRET`     | Secret used to sign admin sessions.           |
| `JWT_EXPIRES_IN` | Session lifetime, e.g. `7d`.                  |

See **[DEPLOYMENT.md](./DEPLOYMENT.md)** for step-by-step Hostinger deployment instructions.

## Writing blog posts

Blog content is written in **Markdown** (bold, lists, links, headings, etc.) directly in the admin editor,
in both English and French. The public article page renders it with proper styling.
